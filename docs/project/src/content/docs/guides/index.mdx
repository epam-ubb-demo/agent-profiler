## What is Agent Profiler?

Agent Profiler is a desktop application that captures OpenTelemetry traces from GitHub Copilot CLI sessions and renders them as interactive timeline views. It gives you span-level visibility into every tool call, model invocation, and permission check across your AI agent sessions — helping you understand latency, token usage, and agent behaviour at a glance.

## Prerequisites

Before you begin, ensure you have:

- **Node.js 20+** and **pnpm** installed
- **GitHub Copilot CLI** (`gh copilot`) installed and authenticated
- Access to the [OTel Gateway demo endpoint](/agent-profiler/tool/otel-gateway/) or your own collector

## Quick Setup

<Steps>
1. **Enable telemetry export in the Copilot CLI**

   Add the following lines to your `~/.zshrc` or `~/.bashrc`:

   ```bash
   export COPILOT_OTEL_ENABLED=true
   export OTEL_EXPORTER_OTLP_ENDPOINT=https://ca-ghcp-demo-uksouth-001.braveglacier-03fb7d47.uksouth.azurecontainerapps.io
   ```

   Reload your shell:

   ```bash
   source ~/.zshrc
   ```

2. **Run a Copilot CLI session**

   ```bash
   gh copilot suggest "list all files in the current directory"
   ```

3. **Verify spans are arriving**

   Check the collector is healthy:

   ```bash
   curl -s -o /dev/null -w "%{http_code}" \
     https://ca-ghcp-demo-uksouth-001.braveglacier-03fb7d47.uksouth.azurecontainerapps.io
   # Expected: 200
   ```

   Then query Application Insights to confirm your spans landed:

   ```kusto
   dependencies
   | where timestamp > ago(15m)
   | project timestamp, name, duration
   | order by timestamp desc
   | take 10
   ```

4. **Download and open Agent Profiler**

   Download the latest release for your platform from the [GitHub Releases page](https://github.com/epam-ubb-demo/agent-profiler/releases) and install it.

   On first launch, configure a data source pointing to the OTel Gateway or your Application Insights workspace.
</Steps>
