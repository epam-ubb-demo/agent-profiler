The Agent Profiler project documentation site is built with [Astro](https://astro.build/) and [Starlight](https://starlight.astro.build/). All project documentation lives in `docs/project/src/content/docs/` within the monorepo.

## File Format

All documentation pages use the `.mdx` extension (MDX = Markdown + JSX). Every page must begin with a frontmatter block:

```mdx
---
title: My Page Title
description: A concise one-sentence description for search engines and SEO.
---

Page content begins here.
```

Both `title` and `description` are **required**.

## Directory Structure

Pages are organised by section under `docs/project/src/content/docs/`:

```
docs/project/src/content/docs/
├── index.mdx              # Project site home page
├── guides/                # Getting started and tutorials
├── decisions/             # Architecture Decision Records
├── operations/            # Release runbooks and ops guides
└── contributing/          # Contributor guides (you are here)
```

Each directory should have an `index.mdx` that introduces the section and links to its pages.

For the tool documentation site (OTel Gateway, Desktop App, Adapters), see `docs/tool/src/content/docs/`.

## Internal Links

Internal links must include the full base path prefix `/agent-profiler/project/`:

```mdx
<!-- Correct -->
See [Architecture Decisions](/agent-profiler/project/decisions/) for design rationale.

<!-- Incorrect — missing base path and site prefix -->
See [Architecture Decisions](/decisions/) for design rationale.
```

For links to the tool documentation site, use the full base path `/agent-profiler/tool/`:

```mdx
See [OTel Gateway](/agent-profiler/tool/otel-gateway/) for setup details.
```

Starlight's links validator plugin validates internal links at build time. Cross-site links (to `/agent-profiler/tool/`) are excluded from validation by configuration.

## Adding a New Page

1. Create a `.mdx` file in the appropriate section directory.
2. Add the required frontmatter (`title` and `description`).
3. Update the parent section's `index.mdx` to link to the new page.
4. Run the build locally to verify there are no broken links.

## Adding a New ADR

ADRs live in `docs/project/src/content/docs/decisions/`. Use the naming convention:

```
adr-<NNNN>-<short-slug>.mdx
```

After creating the ADR file, add a row to the table in [`decisions/index.mdx`](/agent-profiler/project/decisions/).

## Building Locally

```bash
cd docs/project
npm run dev     # Start the dev server with hot reload
npm run build   # Production build (runs the links validator)
```

The dev server starts at `http://localhost:4321/agent-profiler/project/`.

## Style Guide

- Write in **British English** (colour, organisation, behaviour, summarise, artefact).
- Use sentence case for headings.
- Prefer tables for structured reference data.
- Use [Starlight admonitions](https://starlight.astro.build/guides/authoring-content/#asides) (`:::tip`, `:::caution`, `:::note`) for callouts.
- Code blocks should specify the language for syntax highlighting.
- Use `:::note[Coming Soon]` for stub pages that are not yet fully written.
