## Release Overview

Agent Profiler uses [Changesets](https://github.com/changesets/changesets) for versioning. The typical flow is:

1. Feature PRs include a changeset file describing the change.
2. The `version.yml` workflow opens a "Version Packages" PR automatically.
3. Merge the version PR to create tags.
4. The `release.yml` workflow builds platform installers on tag push.

For the full step-by-step process, start with the [End-to-End Runbook](/agent-profiler/operations/release-end-to-end/).
