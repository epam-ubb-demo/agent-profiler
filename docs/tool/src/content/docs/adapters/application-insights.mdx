## Overview

The Application Insights adapter connects Agent Profiler directly to an Azure Application Insights workspace. It queries the `dependencies` table using KQL to retrieve spans collected by the [OTel Gateway](/agent-profiler/tool/otel-gateway/) and converts them into `Session` objects.

## When to Use This Adapter

Use the Application Insights adapter when:

- You want to browse historical sessions collected from multiple developer machines.
- You do not have direct access to the local `.jsonl` log files.
- You need to query spans across a time range or by session ID.

For real-time local sessions, the [Copilot CLI adapter](/agent-profiler/tool/adapters/copilot-cli/) reading directly from log files is faster.

## Prerequisites

- An Azure Application Insights resource with spans ingested via the [OTel Gateway](/agent-profiler/tool/otel-gateway/).
- An Azure identity (user or managed identity) with the **Monitoring Reader** role on the Application Insights resource.

## Configuration

Connection details will be configured in **Settings → Data Sources → Application Insights**:

| Setting | Description |
|---|---|
| Subscription ID | Azure subscription containing the Application Insights resource |
| Resource Group | Resource group name |
| Application Insights Name | Resource name |
| Lookback window | How far back to query (e.g., 7 days) |
